It says:
Changed in version 4.0: The callback argument is now optional and a Future will be returned if it is omitted.
This is true if both callback=None and streaming_callback=None, but not when streaming_callback is not None. For example:
p = Subprocess([...], stdout=Subprocess.STREAM)
# raise exception
yield p.stdout.read_until_close(streaming_callback=print)
# need to wrap in Task
yield Task(p.stdout.read_until_close, streaming_callback=print)
Is this an intended behavior?
