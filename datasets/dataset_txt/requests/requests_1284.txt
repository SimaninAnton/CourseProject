In version 2.7.0 I was using Requests constructor with json parameter specified, like:
json_data = [ ... ];
r = requests.Request('POST', url, json=json_data)
And in the 2.8.0 version this code does not work, because I do not specify data parameter. Inside constructor we get self.data assigned as [] if data is None else data, so we finally get self.data equal to [].
And because of this commit: 7a0cd16 we get json parameter ignored, because now we have to explicitly specify data={} when calling constructor, like:
json_data = [ ... ];
r = requests.Request('POST', url, json=json_data, data={})
I think that this issue definitely breaks documented API.
