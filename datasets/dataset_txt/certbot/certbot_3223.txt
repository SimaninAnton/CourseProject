Using v0.0.0.dev20151020 and its letsencrypt-auto script on Ubuntu 14.04.3 LTS, installation fails because of subj.
Adding --pre as suggested in $HOME/.pip/pip.log like so fixes the issue:
  # nginx is buggy / disabled for now...
  $VENV_BIN/pip install --pre -U letsencrypt > /dev/null
  echo -n .
  $VENV_BIN/pip install --pre -U letsencrypt-apache > /dev/null
