I think it's a good idea to add start parameter in the LabelEncoder like the snippet below. Something after I encoded the label and feed it into a statistical transformation (boxcox) that doesn't accept zero and negative numbers I need to transform the zero's into positive numbers.
>>> from sklearn import preprocessing
>>> le = preprocessing.LabelEncoder(start=1)
>>> le.fit([1, 2, 2, 6])
LabelEncoder()
>>> le.classes_
array([1, 2, 6])
>>> le.transform([1, 1, 2, 6]) 
array([1, 1, 2, 3]...)
>>> le.inverse_transform([1, 1, 2, 3])
array([1, 1, 2, 6])
