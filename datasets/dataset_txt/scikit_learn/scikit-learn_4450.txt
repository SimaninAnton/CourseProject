The jaccard_similarity_score is I believe in error for binary arrays. Using sklearn 0.14.1:
sklearn.metrics.jaccard_similarity_score(np.array([1,1]), np.array([1,1]))
1.0  # correct

sklearn.metrics.jaccard_similarity_score(np.array([1,1]),np.array([1,0]))
0.5  # correct

sklearn.metrics.jaccard_similarity_score(np.array([1,1,0]),np.array([1,0,0]))
0.66666666666666663  # incorrect, should be 0.5

sklearn.metrics.jaccard_similarity_score(np.array([0]), np.array([0]))
1.0  # incorrect, should be 0
Pairs of 0s should not influence the similarity score.
Passing in a 1D array of binary variables causes _check_clf_targets to set y_type=='binary', it looks as though the third calculation method is used:
https://github.com/scikit-learn/scikit-learn/blob/master/sklearn/metrics/metrics.py#L1149
and this looks for the similarity between pairs of 1s or pairs of 0s without respect to the cardinality of the true members of the two arrays.

http://en.wikipedia.org/wiki/Jaccard_similarity#Similarity_of_asymmetric_binary_attributes
