It seems that arbitrary pairwise function can now be passed to SVC to compute the kernel function between 2 samples.
For instance I tried:
In [2]: from sklearn.metrics.pairwise import rbf_kernel

In [3]: from sklearn.svm import SVC

In [4]: SVC(kernel=rbf_kernel)
Out[4]: 
SVC(C=1.0, cache_size=200, class_weight=None, coef0=0.0, degree=3, gamma=0.0,
  kernel=<function rbf_kernel at 0x103e40578>, probability=False,
  shrinking=True, tol=0.001, verbose=False)

In [5]: svm = SVC(kernel=rbf_kernel)

In [6]: from sklearn.datasets import load_iris

In [7]: iris = load_iris()

In [8]: svm.fit(iris.data, iris.target)
Out[8]: 
SVC(C=1.0, cache_size=200, class_weight=None, coef0=0.0, degree=3, gamma=0.0,
  kernel=<function rbf_kernel at 0x103e40578>, probability=False,
  shrinking=True, tol=0.001, verbose=False)

In [9]: svm.score(iris.data, iris.target)
Out[9]: 0.98666666666666669
This feature is not documented in the docstring:
http://scikit-learn.org/dev/modules/generated/sklearn.svm.SVC.html
Actually: the narrative documentation is already up-to-date:
