Documentation at https://keras.io/preprocessing/image/
gives a list of valid values for "class_mode" as [" categorical", "binary", "sparse", '"None"]
But if an invalid value is given a ValueError exception is raised and the list of valid values include all of the above but in addition "input". Either the documentation needs to be revised or "input" needs to be removed for the ValueError message.
To reproduce this, after relevant import enter the following
x = g.flow_from_directory(class_mode='Foo-Bar', directory='/data')
You will get the following:
raise ValueError('Invalid class_mode:', class_mode, '; expected one of "categorical", ' '"binary", "sparse", "input"' ' or None.')
