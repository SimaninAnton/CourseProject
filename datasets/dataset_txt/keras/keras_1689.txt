One cannot create a bool variable (or setting a bool value) when using the tensorflow backend. E.g.
import keras.backend as K
K.variable(False, dtype='bool')
# ValueError: ('Unsupported dtype:', 'bool')
import tensorflow as tf
import keras.backend as K
v = tf.get_variable('variable', False, dtype=tf.bool)
K.set_value(v, True)
# ValueError: ('Unsupported dtype:', 'bool')
This seems to be due to mapping in keras.backend.tensorflow_backend._convert_string_dtype missing an item for bool (link). Is there a reason not to add bool? Furthermore, it seems to me that an even better option would be to simply use tf.as_dtype(x). Is there a reason not to do so?
I'd happily contribute any of these changes if deemed reasonable.
