I don't know if this is an already documented issue, but when a project has a spider with a non-ASCII name, scrapy crawl fails with error KeyError: 'Spider not found: ara\xc3\xb1a'
The following fails with Python 2 (but works with Python 3):
$ scrapy version -v
Scrapy    : 1.1.2
lxml      : 3.6.4.0
libxml2   : 2.9.4
Twisted   : 16.4.0

pyOpenSSL : 16.1.0 (OpenSSL 1.0.2g-fips  1 Mar 2016)
Platform  : Linux-4.4.0-36-generic-x86_64-with-Ubuntu-16.04-xenial


$ scrapy list
araña

$ cat unicodename/spiders/example.py
# -*- coding: utf-8 -*-
import scrapy


class ExampleSpider(scrapy.Spider):
    name = u"araña"
    allowed_domains = ["example.com"]
    start_urls = (
        'http://www.example.com/',
    )

    def parse(self, response):
        yield {"url": response.url}
