Issue
When running scrapy crawl, item fields with Unicode characters can be quite unreadable, for example:
u'["M\u0117sa, \u017euvys ir kulinarija", "\u0160vie\u017eia pauk\u0161tiena"]
instead of
u'["Mėsa, žuvys ir kulinarija", "Šviežia paukštiena"]'
Cause
scrapy/scrapy/item.py
Line 94 in 1fd1702
 return pformat(dict(self)) 
pformat in Python 2 behaves differently than in Python 3.
Python 2:
In: pformat(u'å')
Out: "u'\\xe5'"
Python 3:
In: pformat('å')
Out: "'å'"
Suggested solution
Overriding PrettyPrinter.format:
https://stackoverflow.com/a/10883893/8981993
