When custom middleware in DOWNLOADER_MIDDLEWARES is added below all redirect-related middlewares in chain, custom's process_response method is not executed before redirect-related ones.
I tried to override chain directly in project's settings and extensions without any results. Setting redirect middlewares to None works as expected.
scrapy_redirect_first.py:
import scrapy

class DummyDownloaderExtension(object):
    def process_response(self, request, response, spider):
        if 'something' in response.url:
            request.meta['dont_redirect'] = True

        print("==============> DUMMY YOU!")
        return response

class Spider(scrapy.Spider):
    name = 'github-scrapy'
    start_urls = ['http://blog.scrapinghub.com'] # use http, to be redirected to https
    custom_settings = {
        'DOWNLOADER_MIDDLEWARES': {
            'scrapy_redirect_first.DummyDownloaderExtension': 50,
            'scrapy.downloadermiddlewares.redirect.MetaRefreshMiddleware': 580,
            'scrapy.downloadermiddlewares.redirect.RedirectMiddleware': 600
        }
    }

    def parse(self, response):
        return {}
