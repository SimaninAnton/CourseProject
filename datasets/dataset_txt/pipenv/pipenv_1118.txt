The [scripts] section of the Pipfile does not support shell env variables. For example, all the following commands fails to run correctly:
[scripts]
generate_key = "python generate_key.py ~"
generate_key = "python generate_key.py $HOME"
generate_key = "python generate_key.py ${HOME}"
Expected result
If the absolute path is specified, the script works perfectly:
[scripts]
generate_key = "python generate_key.py /Users/user"
Actual result
pipenv should interpret the variables (e.g. ~, $HOME) and replace them accordingly. For example, if you run echo ~ or echo "$HOME" in the shell you will get your user directory not the literal strings (~ or $HOME).
