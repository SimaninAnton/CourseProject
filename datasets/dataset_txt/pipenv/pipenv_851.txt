Issue description
For git urls, pipenv lock stores commit hash instead of git ref to Pipfile.lock. pipenv lock -r then creates invalid lines.
Expected result
-i https://pypi.org/simple
git+https://github.com/dfunckt/django-rules@master#egg=django-rules
Actual result
-i https://pypi.org/simple
git+https://github.com/dfunckt/django-rules@ec0d0905240cd1e4d79724fd53f7fd461023a3bf#egg=django-rules
Steps to replicate
pipenv install "git+https://github.com/dfunckt/django-rules@master#egg=django-rules"
pipenv lock -r
$ pipenv --support
