When a package is installed pipenv rewrites the Pipfile but doesn't retain line endings. This test is in Linux but I noticed it first in Windows. We keep lines with LF in Git but Pipenv changed it to CRLF. I would think that matching existing line endings and using native for a new file would be proper, but for me it'd be fine to just always use LF.
