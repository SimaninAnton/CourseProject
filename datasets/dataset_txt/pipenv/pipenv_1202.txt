When the pipfile includes private PyPi's, pipenv lock -r should output the index url with the requirement.
Here's an example:
craig@amogato2:~/src/boughtbymany/private_pypi_lock
$ pipenv lock -r
click==6.7
my-python-project==0.1.0

craig@amogato2:~/src/boughtbymany/private_pypi_lock
$ cat Pipfile

craig@amogato2:~/src/boughtbymany/private_pypi_lock
$ pipenv run pip install my-python-project==0.1.0
Collecting my-python-project==0.1.0
  Could not find a version that satisfies the requirement my-python-project==0.1.0 (from versions: )
No matching distribution found for my-python-project==0.1.0

craig@amogato2:~/src/boughtbymany/private_pypi_lock
$ cat Pipfile
[[source]]
url = "https://pypi.python.org/simple"
verify_ssl = true
name = "pypi"

[[source]]
url = "https://testpypi.python.org/simple"
verify_ssl = true
name = "test_pypi"

[packages]
my_python_project = {version = "==0.1.0", index = "test_pypi"}

[dev-packages]

[requires]
python_version = "2.7"
$ python -m pipenv.help output
Expected result
$ pipenv lock -r
click==6.7
my-python-project==0.1.0 --extra-index-url https://testpypi.python.org/simple
Actual result
$ pipenv lock -r
click==6.7
my-python-project==0.1.0

Use Pipfile above
Run pipenv lock -r
