If you encounter a "no module named libmproxy" error, your code is probably based on an old version of mitmproxy. As mentioned in the CHANGELOG for mitmproxy 0.17, you now have to import from mitmproxy instead of libmproxy.
(documenting this here as per https://twitter.com/botnet_hunter/status/740770513690230784)
