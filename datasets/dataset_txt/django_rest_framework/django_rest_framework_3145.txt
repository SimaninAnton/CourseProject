If you are using example ugettext_lazy from django.utils.translation in some output via the djangorestframework.views.View handler, you will get the "TypeError: <django.utils.functional.proxy object at 0x102ef8250> is not JSON serializable" error message.
A solution which I have used on another project is something like this:
from django.utils.translation import ugettext_lazy as _

class LazyEncoder(simplejson.JSONEncoder):
    def default(self, obj):
        if isinstance(obj, Promise):
            return force_unicode(obj)
        return super(LazyEncoder, self).default(obj)

simplejson.dumps({'data': 'normal text', 'translated': _('translated text')}, cls=LazyEncoder)
