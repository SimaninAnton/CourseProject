Considering the following yaml output:
{daps: 'http://127.0.0.1:8000/api/daps/', metadaps: 'http://127.0.0.1:8000/api/metadaps/',  users: 'http://127.0.0.1:8000/api/users/'}
While the first two URLs works fine, the last one is interpreted as follows:
...users: &#39;<a href="http://127.0.0.1:8000/api/users/&#39;%7D" rel="nofollow">http://127.0.0.1:8000/api/users/&#39;}</a>
Instead of:
...users: &#39;<a href="http://127.0.0.1:8000/api/users/" rel="nofollow">http://127.0.0.1:8000/api/users/</a>&#39;}
(The ending quote and brackets are in the link.) Happens basically on any last URL in a bracket, except sometimes the bracket is in the link and sometimes only the ' sign.
dap_set: ['http://127.0.0.1:8000/api/daps/2/']
Goes to:
dap_set: [&#39;<a href="http://127.0.0.1:8000/api/daps/2/&#39;" rel="nofollow">http://127.0.0.1:8000/api/daps/2/&#39;</a>]
Instead if:
dap_set: [&#39;<a href="http://127.0.0.1:8000/api/daps/2/" rel="nofollow">http://127.0.0.1:8000/api/daps/2/</a>&#39;]

REST_FRAMEWORK = {
    # ...
    'DEFAULT_RENDERER_CLASSES': (
        'rest_framework.renderers.YAMLRenderer',
