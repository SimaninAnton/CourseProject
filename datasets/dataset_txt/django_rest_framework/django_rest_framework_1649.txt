It would be much more convenient for mixin based view construction if the perform_create/perform_update/etc methods allowed subclasses to build up the args/kwargs passed to save()
Something like the following instead of https://github.com/tomchristie/django-rest-framework/blob/master/rest_framework/mixins.py#L25:
class CreateModelMixin(object):
    ...
    def perform_create(self, serializer):
        args = self.perform_create_args(serializer)
        kwargs = self.perform_create_kwargs(serializer)
        serializer.save(*args, **kwargs)

    def perform_create_args(self, serializer):
        return []

    def perform_create_kwargs(self, serializer):
        return {}
    ...
for each of the relevant methods
