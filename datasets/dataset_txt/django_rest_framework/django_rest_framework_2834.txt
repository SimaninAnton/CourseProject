Invalid SlugFields are not being caught by validation. I think this is because validators.validate_slug is not specified as a default validator in the class definition.
SlugField Class definition in rest_framework.fields:
class SlugField(CharField):
    type_name = 'SlugField'
SlugField class definition in django.forms.fields:
class SlugField(CharField):
    default_error_messages = {
        'invalid': _(u"Enter a valid 'slug' consisting of letters, numbers,"
                     u" underscores or hyphens."),
    }
    default_validators = [validators.validate_slug]
Compare this to EmailField. First in rest_framework.fields:
class EmailField(CharField):
    type_name = 'EmailField'
    form_field_class = forms.EmailField

    default_error_messages = {
        'invalid': _('Enter a valid e-mail address.'),
    }
    default_validators = [validators.validate_email]
and in django.forms.fields:
class EmailField(CharField):
    default_error_messages = {
        'invalid': _(u'Enter a valid e-mail address.'),
    }
    default_validators = [validators.validate_email]
