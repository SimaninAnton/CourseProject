I'm not sure if this is by design, but you can't set a value for slug_url_kwarg on a subclass of HyperlinkedRelatedField and have it be honored.
The key is in the __init__ for HyperlinkedRelatedField:
...
self.slug_url_kwarg = kwargs.pop('slug_url_kwarg', self.slug_field)
...
This precludes writing something like:
class WeirdCustomHyperlinkedRelatedField(serializers.HyperlinkedRelatedField):
    slug_field = 'username'
    slug_url_kwarg = 'slug'
    ...
Please advise.
