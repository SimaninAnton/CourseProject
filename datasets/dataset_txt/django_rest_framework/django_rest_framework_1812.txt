You can set allow_null=True on a CharField, but None gets converted to a string before going to the validator. I believe the problem is the to_internal_value() function, which coerces None to a unicode string:
https://github.com/tomchristie/django-rest-framework/blob/master/rest_framework/fields.py#L588:
    def to_internal_value(self, data):
        value = six.text_type(data)
        return value.strip() if self.trim_whitespace else value
If you run this on None, you just a get the string u'None':
>>> from six import text_type
>>> text_type(None)
u'None'
>>> 
I suggest changing it like so:
    def to_internal_value(self, data):
        if data is None and self.allow_null:
            return None
        value = six.text_type(data)
        return value.strip() if self.trim_whitespace else value
